# 선수 간 조합 분석 기획서

## 배경

축구는 11대 11의 팀 스포츠로, 개별 선수의 능력보다 선수 간 조합과 연계가 더 중요합니다.
현재 개별 선수 분석만으로는 팀의 실제 성과를 완전히 설명하기 어렵습니다.

## 분석 목표

1. **효과적인 선수 조합 식별**: 어떤 선수들이 함께 뛸 때 시너지가 발생하는가?
2. **롤 조합 최적화**: 특정 롤 조합이 팀 성과에 어떤 영향을 미치는가?
3. **패스 네트워크 분석**: 선수 간 패스 연결 패턴과 그 효과
4. **공간 활용 조합**: 선수들의 위치 조합이 공격/수비에 미치는 영향

## 분석 방법

### 1. 패스 네트워크 분석

**목적**: 선수 간 패스 연결의 빈도, 성공률, 패턴 분석

**지표**:
- 선수 A → 선수 B 패스 빈도
- 선수 A → 선수 B 패스 성공률
- 패스 네트워크 중심성 (어떤 선수가 핵심 연결고리인가)
- 패스 경로 다양성 (직접 패스 vs 간접 패스)

**활용**:
- 핵심 연결 선수 식별
- 패스 경로 최적화 제안
- 특정 선수 조합의 패스 효율성 평가

### 2. 롤 조합 분석

**목적**: 특정 롤 조합이 팀 성과에 미치는 영향 분석

**분석 방법**:
- 같은 경기에 출전한 선수들의 롤 조합
- 각 롤 조합별 팀 승률, 득점, 실점 비교
- 롤 조합의 균형성 평가 (예: 공격형 롤 vs 수비형 롤)

**예시**:
- "Deep Lying Playmaker + Box-to-Box Midfielder" 조합의 효과
- "Ball Playing Defender + No-Nonsense Centre-Back" 조합의 효과

### 3. 공간 활용 조합

**목적**: 선수들의 평균 위치 조합이 공격/수비에 미치는 영향

**지표**:
- 선수들의 평균 터치 위치 분포
- 공간 커버리지 (선수들이 피치를 얼마나 잘 커버하는가)
- 공간 중복도 (선수들이 같은 공간을 사용하는가)

**활용**:
- 공간 활용 최적화 제안
- 전술적 약점 식별 (공간이 비는 구역)

### 4. 시너지 효과 분석

**목적**: 특정 선수 조합이 함께 뛸 때의 시너지 효과 측정

**방법**:
- 선수 A와 B가 함께 뛴 경기 vs 따로 뛴 경기의 성과 비교
- 선수 조합별 팀 승률, 득점, 실점 분석
- 선수 조합의 상호 보완성 평가

**예시**:
- "공격형 미드필더 + 수비형 미드필더" 조합
- "빌드업형 센터백 + 딥라잉 플레이메이커" 조합

## 구현 계획

### Phase 1: 패스 네트워크 분석
1. 선수 간 패스 연결 매트릭스 생성
2. 패스 네트워크 시각화
3. 핵심 연결 선수 식별
4. 패스 경로 효율성 분석

### Phase 2: 롤 조합 분석
1. 경기별 선수 롤 조합 추출
2. 롤 조합별 성과 지표 계산
3. 효과적인 롤 조합 식별
4. 팀별 최적 롤 조합 제안

### Phase 3: 공간 활용 조합
1. 선수들의 평균 위치 매핑
2. 공간 커버리지 분석
3. 공간 중복도 계산
4. 공간 활용 최적화 제안

### Phase 4: 시너지 효과 분석
1. 선수 조합별 출전 경기 추출
2. 조합별 성과 지표 계산
3. 시너지 효과 측정
4. 최적 선수 조합 제안

## 예상 결과물

1. **패스 네트워크 시각화**: 선수 간 패스 연결 그래프
2. **롤 조합 효과 분석 리포트**: 효과적인 롤 조합과 그 이유
3. **공간 활용 분석 리포트**: 팀의 공간 활용 패턴과 개선 방안
4. **시너지 효과 분석 리포트**: 최적 선수 조합과 시너지 효과

## 활용 방안

1. **전술 분석**: 감독이 선수 조합을 최적화하는 데 활용
2. **선수 영입**: 팀에 필요한 롤/선수 타입 식별
3. **선수 배치**: 최적의 선수 조합으로 라인업 구성
4. **전술 개선**: 공간 활용과 패스 경로 최적화

## 기술적 고려사항

1. **데이터 요구사항**:
   - 경기별 선수 출전 정보
   - 선수 간 패스 연결 데이터
   - 선수들의 위치 데이터

2. **계산 복잡도**:
   - 선수 조합 수: nC2 (n명의 선수)
   - 경기 수가 많을수록 계산량 증가
   - 효율적인 알고리즘 필요

3. **검증 방법**:
   - 실제 경기 결과와 비교
   - 도메인 전문가 검토
   - 통계적 유의성 검증
